Respuestas

  1. ¿Qué es el DOM?
    El DOM (Document Object Model) es una representación en forma de árbol de los elementos que conforman una página web, como HTML, XML o XHTML. Permite a los desarrolladores interactuar y manipular los elementos de la página a través de lenguajes de programación como JavaScript.
  2. ¿Qué es un nodo?
    Un nodo es un objeto individual que forma parte del árbol del DOM. Cada elemento, atributo y texto en el documento es un nodo. Los nodos pueden ser de diferentes tipos, como nodos de elementos, nodos de texto y nodos de atributos.
  3. ¿Cómo se accede a un elemento del DOM?
    Se puede acceder a un elemento del DOM utilizando métodos de selección de JavaScript como getElementById, getElementsByClassName, getElementsByTagName o querySelector y querySelectorAll. Estos métodos devuelven referencias a los elementos en el DOM que coinciden con los criterios especificados.
  4. ¿Cómo se modifica un elemento del DOM?
    Para modificar un elemento del DOM, primero se debe acceder a él utilizando uno de los métodos de selección mencionados anteriormente. Luego, se pueden cambiar sus atributos, contenido o estilo utilizando propiedades y métodos de JavaScript como innerHTML, textContent, setAttribute, style, etc.
  5. ¿Cuál es la diferencia entre getElementBy* y querySelector*?
    La principal diferencia entre getElementBy* (como getElementById, getElementsByClassName, getElementsByTagName) y querySelector* (querySelector y querySelectorAll) es la sintaxis y la flexibilidad. Los métodos getElementBy* buscan elementos basados en un solo criterio (ID, clase o etiqueta). Por otro lado, querySelector* permite buscar elementos usando selectores de CSS, lo que proporciona mayor flexibilidad al seleccionar elementos. Además, querySelector devuelve el primer elemento que coincide con el selector, mientras que querySelectorAll devuelve todos los elementos que coinciden con el selector.
  6. ¿Qué es un evento?
    Un evento es una acción o suceso que ocurre en el navegador, como un clic del mouse, el presionar una tecla, la carga de una página, etc. Los eventos permiten a los desarrolladores ejecutar código de JavaScripten respuesta a dichas acciones, proporcionando una mayor interacción y dinamismo en las páginas web.
  7. ¿Qué es un listener?
    Un listener (o escuchador) es una función en JavaScript que espera y responde a un evento específico. Cuando se dispara el evento, el listener se ejecuta, permitiendo llevar a cabo acciones en respuesta al evento. Los listeners se pueden agregar a elementos del DOM utilizando métodos como addEventListener.
  8. ¿Qué es un callback?
    Un callback es una función que se pasa como argumento a otra función. Esta función es llamada (o "devuelta") en algún punto de la ejecución de la función principal. Los callbacks son comunes en JavaScript, especialmente en operaciones asíncronas, y permiten ejecutar código después de que se haya completado una tarea específica, como la carga de un recurso o el disparo de un evento.
  9. ¿Qué es un objeto de evento?
    Un objeto de evento es un objeto que se crea automáticamente cuando se dispara un evento. Este objeto contiene información sobre el evento, como el tipo de evento, la hora en que ocurrió, el elemento que lo generó, entre otros datos. El objeto de evento se pasa como argumento a la función listener que se ejecuta en respuesta al evento, permitiendo a los desarrolladores acceder a la información del evento y realizar acciones específicas según las características del evento.
